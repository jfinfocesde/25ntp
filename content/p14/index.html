    Semana 14: Despliegue de Aplicaciones Streamlit en la Nube

    Objetivos de la Semana

    • Aprender a desplegar aplicaciones Streamlit en servicios cloud gratuitos
    • Comprender los conceptos básicos de despliegue web
    • Configurar repositorios Git para despliegue automático
    • Resolver problemas comunes en el despliegue

    Introducción al Despliegue en la Nube

    El despliegue en la nube permite que tus aplicaciones Streamlit estén disponibles 24/7 en internet sin necesidad de mantener tu computadora encendida. Existen varias plataformas que ofrecen servicios gratuitos perfectos para proyectos educativos y prototipos.

    Opciones de Despliegue Gratuito

    1. Streamlit Community Cloud (Recomendado)

    Ventajas:

    • Diseñado específicamente para Streamlit
    • Despliegue automático desde GitHub
    • Sin límite de tiempo de ejecución
    • Soporte para dependencias personalizadas

    Requisitos:

    • Cuenta de GitHub
    • Repositorio público con tu aplicación

    2. Heroku

    Ventajas:

    • Soporte para múltiples lenguajes
    • Buena documentación
    • Dyno gratuito disponible

    Limitaciones:

    • Se duerme después de 30 minutos de inactividad
    • Límite de horas mensuales

    3. Railway

    Ventajas:

    • Interfaz moderna
    • Despliegue desde GitHub
    • Buen rendimiento

    Limitaciones:

    • Crédito mensual limitado en plan gratuito

    Tutorial: Despliegue en Streamlit Community Cloud

    Paso 1: Preparar tu Aplicación

    1. Estructura del proyecto: 

      mi-app-streamlit/
├── app.py              # Archivo principal
├── requirements.txt    # Dependencias
├── README.md          # Documentación
└── .gitignore        # Archivos a ignorar

    2. Crear requirements.txt: 

      streamlit>=1.28.0
pandas>=1.5.0
numpy>=1.24.0
# Agrega aquí todas las librerías que uses

    3. Configurar app.py: 

      import streamlit as st
import pandas as pd
import numpy as np

# Configuración de la página
st.set_page_config(
    page_title="Mi Aplicación Streamlit",
    page_icon="🚀",
    layout="wide"
)

st.title("¡Hola desde la nube! 🌤️")
st.write("Esta aplicación está desplegada en Streamlit Community Cloud")

# Tu código aquí

    Paso 2: Subir a GitHub

    1. Crear repositorio en GitHub:

      • Ve a github.com
      • Crea un nuevo repositorio público
      • No inicialices con README (ya tienes uno local)

    2. Subir tu código: 

      git init
git add .
git commit -m "Primera versión de mi app Streamlit"
git branch -M main
git remote add origin https://github.com/TU_USUARIO/TU_REPOSITORIO.git
git push -u origin main

    Paso 3: Desplegar en Streamlit Community Cloud

    1. Crear cuenta:

      • Ve a share.streamlit.io
      • Inicia sesión con tu cuenta de GitHub

    2. Desplegar aplicación:

      • Haz clic en "New app"
      • Selecciona tu repositorio
      • Elige la rama (main)
      • Especifica el archivo principal (app.py)
      • Haz clic en "Deploy"

    3. Esperar el despliegue:

      • El proceso toma 2-5 minutos
      • Recibirás una URL como: https://tu-app.streamlit.app

    Configuración Avanzada

    Variables de Entorno

    Para aplicaciones más complejas, puedes usar variables de entorno:

    import os
import streamlit as st

# En tu app.py
API_KEY = os.getenv('API_KEY', 'valor_por_defecto')
st.write(f"API Key: {API_KEY}")

    Configurar en Streamlit Cloud:

    1. Ve al dashboard de tu app
    2. Haz clic en "Settings"
    3. En "Secrets", agrega:
    API_KEY = "tu_clave_secreta"

    Manejo de Errores

    import streamlit as st
import traceback

try:
    # Tu código principal
    st.title("Mi Aplicación")
    # ... resto del código
except Exception as e:
    st.error(f"Error: {str(e)}")
    st.code(traceback.format_exc())

    Solución de Problemas Comunes

    Error: "No module named 'xxx'"

    Solución: Asegúrate de incluir todas las dependencias en requirements.txt

    Error: "App is taking too long to load"

    Soluciones:

    • Optimiza tu código
    • Usa @st.cache_data para funciones costosas
    • Reduce el tamaño de datasets

    Error: "Memory limit exceeded"

    Soluciones:

    • Procesa datos en chunks
    • Usa muestras de datos grandes
    • Optimiza el uso de memoria

    Ejemplo de Aplicación

    import streamlit as st
import pandas as pd
import plotly.express as px

st.set_page_config(page_title="Análisis de Datos", layout="wide")

# Cargar datos
@st.cache_data
def cargar_datos():
    return pd.read_csv('https://raw.githubusercontent.com/plotly/datasets/master/iris.csv')

df = cargar_datos()

# Sidebar
st.sidebar.header("Filtros")
especie = st.sidebar.selectbox("Seleccionar especie:", df['Name'].unique())

# Main content
st.title("Análisis del Dataset Iris")

# Filtrar datos
filtered_df = df[df['Name'] == especie]

# Mostrar métricas
col1, col2, col3 = st.columns(3)
col1.metric("Total de muestras", len(filtered_df))
col2.metric("Promedio Sepal Length", f"{filtered_df['SepalLength'].mean():.2f}")
col3.metric("Promedio Petal Length", f"{filtered_df['PetalLength'].mean():.2f}")

# Gráfico
fig = px.scatter(filtered_df, x='SepalLength', y='PetalLength', 
                 color='Name', title=f'Dispersión para {especie}')
st.plotly_chart(fig, use_container_width=True)

# Tabla de datos
st.subheader("Datos filtrados")
st.dataframe(filtered_df)
